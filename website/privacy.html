Privacy Policy

Local Model Lab is built around local inference. This policy explains what stays on your computer, which network requests you initiate, and what you can delete.

Effective July 9, 2026Version 1.0
ScopeLocal processingNetwork activityStorageThird partiesSecurityYour choicesChildrenChanges and contact

1. Scope

This Privacy Policy applies to the Local Model Lab desktop application and this project website. Local Model Lab does not require an account and the current release does not operate a cloud inference service, advertising system, analytics service, crash-reporting service, or telemetry endpoint.

2. Information processed locally

Prompts, system instructions, chat messages, model responses, negative prompts, generation settings, generated images, imported model files, and inference logs are processed on your device. The app does not send this content to a Local Model Lab-operated server.

Hardware information including CPU model, logical core count, total RAM, GPU name, and reported VRAM is read locally to show compatibility guidance. This information is not uploaded by Local Model Lab.

3. Network activity you initiate

When you verify or download a catalog model, the desktop app connects to Hugging Face and its content delivery infrastructure. When you install the optional stable-diffusion.cpp backend, the app connects to GitHub and its release infrastructure. Those providers may receive standard network data such as your IP address, request time, requested file, user agent, and related security logs under their own policies.

Connections to A1111, Forge, ComfyUI, or similar image backends are directed to loopback addresses on your computer. If you separately configure those tools to use remote services, their behavior is outside Local Model Lab's control.

4. Files and settings

Local Model Lab stores models, partial downloads, generations, and application settings in operating-system user-data directories. The 18+ safety preference is stored locally. Exported chat files are written only to the location you select. The website uses no advertising cookies and does not require local storage for normal browsing.

5. Third-party models and software

Model files and optional backends are published by independent parties. Their repositories, download hosts, licenses, privacy practices, and security controls are not operated by Local Model Lab. Review the source and license before downloading. Local Model Lab displays repository and license information as a convenience, but the upstream terms control.

6. Security

The Electron renderer uses an allowlisted IPC bridge with Node integration disabled. Model downloads are limited to approved HTTPS sources and checked for trusted redirects, non-document response types, and expected file size. Partial downloads are kept for resume. These controls reduce risk but cannot guarantee that a third-party model, local backend, operating system, or device is secure.

Do not include passwords, private keys, regulated records, or other sensitive information in prompts unless you understand and accept the security of your own device and every local backend involved.

7. Your controls

  • Delete a model from Model Library or remove files directly from the models folder.
  • Delete generated images from the generations folder.
  • Clear visible engine logs during the current session.
  • Disable Start with Windows in Settings.
  • Uninstall Local Model Lab and remove its user-data directory to erase remaining local app data.

Because Local Model Lab does not receive your local prompts or outputs, it cannot retrieve, export, correct, or delete those files for you.

8. Children

Local Model Lab is not directed to children. Users must satisfy the minimum age required by applicable law and by any selected model license. Sexual content involving minors is prohibited and subject to a hard local safety block.

9. Changes and contact

Material changes will be posted with an updated effective date. Until a public support email is configured, submit privacy and security requests through the official GitHub repository issue tracker after publication. Do not place personal data, prompts, or sensitive file paths in a public issue.
